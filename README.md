# 视频智能分析处理套件

面向 Windows 的本地视频理解工具，支持关键帧提取、图像分析、语音转录和检索式问答。

快速入口：`快速开始`、`使用流程`、`常见问题`

---

## 项目简介

这个项目提供一套从视频上传到结果查看的完整处理流程。程序通过 Streamlit 提供界面，把视频内容拆解为关键帧、语音文本、分析报告和问答结果，适合用于快速浏览长视频内容、整理重点信息或继续检索历史分析结果。

## 核心能力

- 自动按场景提取关键帧。
- 对关键帧执行视觉模型分析。
- 从视频中提取音频并进行中文语音转录。
- 生成完整分析报告、速读摘要和检索问答结果。
- 支持云端模型与本地模型后端切换。
- 将模型缓存与任务产物统一落盘到项目目录，便于复查和迁移。

## 适合场景

- 快速浏览较长的视频内容，先拿到结构化结果再决定是否细看。
- 对会议、课程、访谈等内容做转录、摘要和重点整理。
- 基于已经生成的分析结果继续提问和回查细节。

## 快速开始

### 方式一：Windows 安装脚本

首次使用建议直接在项目根目录运行安装脚本。

1. 双击 `运行环境配置.bat`。
2. 按提示完成 Python 依赖安装和可选的 API 配置。
3. 安装完成后，根目录会生成 `run.bat`。
4. 双击 `run.bat` 启动程序。
5. 首次启动如果需要下载模型，请保持网络连接稳定。

### 方式二：源码启动

适合已经具备 Python 环境、希望直接以源码方式运行的用户。

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
python run.py
```

### 运行前提

- Windows 10/11（64 位）。
- Python 3.8 及以上。
- 至少 8GB 内存。
- 预留足够磁盘空间用于依赖、模型缓存和分析结果。
- 如使用云端模型，需要配置对应的 API Key。
- 如使用本地模型或 GPU 推理，需要自行准备对应运行环境。

## 使用流程

1. 启动程序并进入 Streamlit 页面。
2. 上传视频，或进入批量处理入口。
3. 选择模型后端与分析方式。
4. 执行关键帧提取、语音转录和图像分析。
5. 在页面查看任务结果、报告、摘要和检索问答内容。

## 项目结构

当前仓库已经完成启动入口与源码目录拆分，根目录保留兼容入口，主程序位于 `video_ai_suite/`。

```text
Frame-by-frame-AI-parser_by_JucieOvo/
├── run.py                  # 根启动入口，转发到正式启动器
├── app.py                  # 根级兼容 Streamlit 入口
├── setup.py                # 安装流程入口
├── requirements.txt        # Python 依赖列表
├── 运行环境配置.bat        # Windows 环境配置脚本
├── video_ai_suite/
│   ├── bootstrap/          # 环境检查、安装、下载、启动
│   ├── backend/            # 任务、缓存、模型调用等后端能力
│   └── streamlit_app/      # Streamlit 正式入口与页面逻辑
├── docs/                   # 文档目录
└── README.md               # 项目说明文档
```

### 运行时目录

程序运行后会在项目目录下生成 `.cache/`，用于保存模型缓存、任务私有目录和部分中间产物。首次运行耗时通常也主要集中在这里。

## 常见问题

<details>
<summary><b>Q: 安装过程很慢怎么办？</b></summary>

优先使用国内镜像安装依赖：

```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

</details>

<details>
<summary><b>Q: 运行完安装脚本后找不到 run.bat？</b></summary>

`run.bat` 不是仓库内预置文件，而是安装流程执行后由程序自动生成的启动脚本。如果没有生成，请重新运行 `运行环境配置.bat`，或直接使用 `python run.py` 启动。

</details>

<details>
<summary><b>Q: 首次启动停留时间很长，是否正常？</b></summary>

通常是正常现象。首次运行可能需要下载模型并初始化缓存，相关文件会写入项目目录下的 `.cache/`。请保持网络连接稳定，并预留足够磁盘空间。

</details>

<details>
<summary><b>Q: API 调用失败或模型不可用怎么办？</b></summary>

优先检查以下几项：

1. 是否已正确配置对应模型服务所需的 API Key。
2. 当前网络是否可访问目标模型服务。
3. 云端账户额度、权限或模型名称是否正确。
4. 本地模型服务是否已经提前启动。

如果使用 DashScope，可先在命令行检查环境变量：

```bash
echo %DASHSCOPE_API_KEY%
```

</details>

<details>
<summary><b>Q: 结果和缓存文件保存在哪里？</b></summary>

当前项目会把主要运行时内容写入项目目录下的 `.cache/`。如果你在界面中进行批量处理、关键帧复用或历史结果检索，也可以从这里继续定位对应任务目录。

</details>
